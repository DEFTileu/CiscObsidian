# 5.4 Module Practice and Quiz

## Что нужно уметь проговорить без подсказки

После Module 5 ты должен уметь быстро объяснить:

1. почему Layer 2 redundancy без STP опасна;
2. как выбирается `root bridge`;
3. как выбираются `root port` и `designated port`;
4. почему другие ports block/discard traffic;
5. чем `STP` отличается от `RSTP`;
6. зачем нужны `PortFast` и `BPDU Guard`.

## Быстрый summary

|Тема|Ключевая мысль|
|---|---|
|Purpose of STP|убирает loops, но сохраняет redundancy|
|Root bridge|switch с lowest BID|
|Root port|лучший путь switch к root|
|Designated port|лучший port на каждом segment|
|Blocked / Alternate|резерв, который не даёт образоваться loop|
|RSTP|быстрее converges|
|PortFast|ускоряет access port startup|
|BPDU Guard|отключает опасный PortFast port при BPDU|

## Мини-проверка себя

Попробуй ответить:

- Что произойдёт, если в Layer 2 loop пойдёт ARP broadcast?
- Почему blocked port в STP полезен?
- Что важнее при root election: priority или MAC?
- Что делает `PortFast`, и почему его нельзя включать на switch-to-switch link?
- Что произойдёт, если BPDU придёт на PortFast port с включённым BPDU Guard?

## Команды для повторения

```bash
show spanning-tree
show spanning-tree vlan 1
show spanning-tree root
show spanning-tree summary
show spanning-tree interface f0/1
show interfaces status err-disabled
```

`show spanning-tree` #ciscoIOScommand
Главная verify-команда по STP: показывает root, roles, states и costs.

`show spanning-tree vlan 1` #ciscoIOScommand
Полезна, когда нужно смотреть конкретный VLAN instance.

`show spanning-tree root` #ciscoIOScommand
Позволяет быстро понять, какой switch сейчас root.

`show spanning-tree summary` #ciscoIOScommand
Показывает краткий обзор spanning tree mode и features.

`show spanning-tree interface f0/1` #ciscoIOScommand
Нужна для проверки роли и состояния конкретного port.

`show interfaces status err-disabled` #ciscoIOScommand
Полезна для PortFast/BPDU Guard troubleshooting.

> [!important] Самая важная формула модуля
> `Lowest BID -> Root Bridge`
>
> `Lowest path cost -> Root Port / Designated Port decision`

> [!tip] Если готовишься быстро
> Сначала выучи термины `BID`, `BPDU`, `root bridge`, `root port`, `designated port`, `alternate port`, а потом уже версии протокола.

> [!success] Итог модуля
> Если ты видишь STP topology и можешь объяснить election, blocked path и смысл PortFast/BPDU Guard, значит тему ты реально понял.
