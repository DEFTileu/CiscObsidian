# 7.3 Configure a DHCPv4 Client

## Router как DHCP client

Иногда сам router не раздаёт addresses, а наоборот получает WAN IPv4 configuration от ISP.

Это типичный SOHO / branch scenario.

![](https://itexamanswers.net/wp-content/uploads/2020/06/2020-06-04_185700.jpg)

В таком случае interface на router настраивается как DHCP client.

## Базовая конфигурация

```bash
interface g0/0/1
ip address dhcp
no shutdown
```

### Что делает каждая команда

`interface g0/0/1` #ciscoIOScommand
Открывает нужный interface для настройки.

`ip address dhcp` #ciscoIOScommand
Говорит router получать IPv4 address автоматически от DHCP server.

`no shutdown` #ciscoIOScommand
Включает interface administratively.

## Что будет видно после выдачи адреса

Cisco IOS обычно покажет сообщение вроде:

```text
%DHCP-6-ADDRESS_ASSIGN: Interface GigabitEthernet0/0/1 assigned DHCP address ...
```

Это значит, что interface успешно получил IPv4 settings.

## Проверка

```bash
show ip interface g0/0/1
```

`show ip interface g0/0/1` #ciscoIOScommand
Показывает IPv4 parameters interface и источник адреса.

Ключевая строка в выводе:

```text
Address determined by DHCP
```

## Home router idea

Домашние routers обычно уже по умолчанию настроены как DHCP clients на WAN side.

То есть когда домашний router подключается к ISP modem, он автоматически запрашивает IPv4 address через DHCP.

Это то, что в GUI часто выглядит как:

- `Automatic Configuration - DHCP`

## Быстрый шаблон

|Сценарий|Что делать|
|---|---|
|Router получает WAN IP от ISP|`ip address dhcp`|
|Нужно проверить источник IP|`show ip interface ...`|
|Interface down|`no shutdown`|

> [!warning] Частая ошибка
> Настроить `ip address dhcp`, но забыть `no shutdown`.

> [!tip] Memory hook
> `LAN side often serves DHCP`
>
> `WAN side often receives DHCP`

> [!success] Если понял тему
> Ты можешь быстро настроить Cisco router как DHCP client и проверить, что WAN interface действительно получил address от ISP.
