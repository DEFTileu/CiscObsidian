## 4.5.3 What did I learn in this module?

Если собрать модуль в одну картину, то логика такая:

- `VLANs` разделяют Layer 2 domains;
- inter-VLAN communication требует Layer 3 device;
- есть три main methods:
  - `Legacy`
  - `Router-on-a-Stick`
  - `Layer 3 Switch with SVIs`
- `Router-on-a-Stick` использует one trunk link и multiple subinterfaces;
- `Layer 3 switch` использует SVIs и обычно лучше масштабируется;
- troubleshooting обычно сводится к проверке VLANs, trunks, access ports, host IP settings и router/L3 switch config.

> [!abstract] Самое важное для запоминания
> - `Different VLANs need routing`
> - `Router-on-a-Stick = trunk + subinterfaces`
> - `Layer 3 switch = SVIs + ip routing`
> - `show interfaces trunk` checks trunk
> - `show ip interface brief` checks subinterfaces/SVIs
> - `encapsulation dot1Q <vlan>` binds subinterface to VLAN

## 4.5.4 Module Quiz – Inter-VLAN Routing

На quiz по этой теме лучше идти с пониманием, а не с набором изолированных команд.

Ты должен уверенно отвечать на вопросы:

- почему two VLANs cannot talk without Layer 3;
- когда использовать `Router-on-a-Stick`;
- когда лучше `Layer 3 switch`;
- где искать проблему, если ping between VLANs fails.

> [!success] Главная идея модуля
> `Segment at Layer 2, reconnect at Layer 3`
