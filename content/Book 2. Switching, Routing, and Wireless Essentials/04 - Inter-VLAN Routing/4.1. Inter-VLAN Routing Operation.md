# 4.1 Inter-VLAN Routing Operation

## 4.1.1 What is Inter-VLAN Routing?

Hosts из одной VLAN не могут общаться с hosts из другой VLAN сами по себе.

Причина простая:

- VLAN делит сеть на отдельные Layer 2 broadcast domains;
- переход между ними требует Layer 3 decision.

Есть три основных варианта inter-VLAN routing:

1. `Legacy Inter-VLAN Routing`
2. `Router-on-a-Stick`
3. `Layer 3 Switch using SVIs`

**SVI (Switched Virtual Interface)** — виртуальный Layer 3 interface для VLAN на switch. #abbreviation

|Method|Смысл|
|---|---|
|`Legacy`|Один physical router interface на каждую VLAN|
|`Router-on-a-Stick`|Один physical router interface + много subinterfaces|
|`Layer 3 Switch`|SVIs на multilayer switch|

## 4.1.2 Legacy Inter-VLAN Routing

В старом варианте router подключался к switch несколькими physical interfaces.

Каждый interface router обслуживал одну VLAN и был default gateway для этой VLAN.

![](https://itexamanswers.net/wp-content/uploads/2020/04/Legacy-Inter-VLAN-Routing.jpg)

Проблема этого метода:

- плохо масштабируется;
- быстро заканчиваются physical interfaces на router;
- на каждую VLAN нужен отдельный port.

> [!warning] Что надо запомнить
> `Legacy inter-VLAN routing` сейчас рассматривается как historical method, а не как нормальная modern design.

## 4.1.3 Router-on-a-Stick Inter-VLAN Routing

Этот метод решает ограничение legacy approach.

Вместо множества physical router ports используется:

- один physical interface;
- trunk link до switch;
- несколько `subinterfaces`.

**subinterface** — логический interface внутри одного physical router interface. #networkterm

![](https://itexamanswers.net/wp-content/uploads/2020/04/Router-on-a-Stick-Inter-VLAN-Routing.gif)

Логика работы:

1. Frame приходит на trunk.
2. Router видит VLAN tag.
3. Traffic попадает в нужную subinterface.
4. Router принимает routing decision.
5. Traffic уходит обратно с новым VLAN tag.

Этот метод подходит для:

- small networks;
- medium networks;
- лабораторий;
- учебных topologies.

Но он не очень хорош для больших сетей.

> [!tip] Как запомнить
> `One link, many VLANs, many subinterfaces`

## 4.1.4 Inter-VLAN Routing on a Layer 3 Switch

Современный способ — использовать **Layer 3 switch**.

**Layer 3 switch** — switch, который умеет и Layer 2 switching, и Layer 3 routing. #networkterm
**multilayer switch** — другое название Layer 3 switch. #networkterm

![](https://itexamanswers.net/wp-content/uploads/2020/04/Inter-VLAN-Routing-on-a-Layer-3-Switch.jpg)

Здесь routing делается через `SVIs`.

Плюсы метода:

- выше скорость;
- меньше latency;
- лучше scalability;
- routing остаётся внутри switch;
- удобно для enterprise campus LAN.

Минус в основном один:

- дороже оборудование.

> [!success] Если понял тему
> Ты уже понимаешь:
> - зачем вообще нужен inter-VLAN routing;
> - чем `Legacy`, `Router-on-a-Stick` и `Layer 3 switch` отличаются;
> - почему Layer 3 switch обычно считается modern solution.
