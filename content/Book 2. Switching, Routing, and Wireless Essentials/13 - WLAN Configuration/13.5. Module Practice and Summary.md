# 13.5 Module Practice and Summary

## Что нужно уметь проговорить без подсказки

После Module 13 ты должен быстро объяснить:

1. как базово настроить wireless router для remote site;
2. почему надо менять default admin settings;
3. как на `WLC` создать базовый WLAN с `WPA2-PSK`;
4. как на `WLC` создать `WPA2-Enterprise` WLAN через `RADIUS`;
5. зачем нужны interface/VLAN mapping и DHCP scope;
6. как искать типичные wireless failures и slow WLAN issues.

## Короткий summary

|Тема|Ключевая мысль|
|---|---|
|Wireless router|одно устройство часто даёт AP + switch + router + DHCP + NAT|
|Basic router setup|сначала меняют default admin/password/IP settings|
|Basic wireless setup|SSID, channel, security mode, passphrase|
|Basic WLC WLAN|создать WLAN profile и включить `WPA2-PSK`|
|Enterprise WLAN|использовать `RADIUS`, `802.1X`, `AES`, VLAN interface|
|Troubleshooting|проверять client, RF, security, DHCP и WLC status|

## Самые важные термины

**WLC (Wireless LAN Controller)** — централизованно управляет AP и WLAN policy. #abbreviation
**RADIUS (Remote Authentication Dial-In User Service)** — сервер для централизованной аутентификации. #abbreviation
**SNMP (Simple Network Management Protocol)** — протокол мониторинга и управления устройствами. #abbreviation
**SSID (Service Set Identifier)** — имя wireless network. #abbreviation
**PSK (Pre-Shared Key)** — общий пароль WLAN. #abbreviation
**802.1X** — framework для enterprise authentication. #networkterm
**DHCP scope** — пул адресации и связанных параметров для конкретной сети. #networkterm
**firmware** — встроенное ПО устройства. #networkterm

## Таблицы для памяти

|Сравнение|Что запомнить|
|---|---|
|Home/remote router vs WLC|single box vs centralized enterprise control|
|WPA2-Personal vs WPA2-Enterprise|shared password vs RADIUS/802.1X|
|SSID visible vs usable WLAN|сеть можно увидеть, но без правильной auth она не заработает|
|Connected vs healthy WLAN|association ещё не гарантирует good performance|

## Быстрые формулы

|Формула|Смысл|
|---|---|
|`Default settings = security risk`|самое первое, что меняют|
|`SSID + security + IP services = usable WLAN`|одного имени сети недостаточно|
|`WLC + AP + policy = enterprise WLAN`|централизация управления|
|`WPA2-Enterprise = RADIUS + 802.1X + AES`|ключевая exam связка|
|`Slow Wi-Fi != only ISP issue`|часто виноваты RF и client density|

> [!important] Самая важная мысль модуля
> В WLAN configuration нужно уметь не только создать `SSID`, но и правильно связать security, addressing и controller/AP infrastructure.

> [!tip] Если повторяешь быстро
> Сначала повтори wireless router flow, потом базовый WLAN на `WLC`, потом `RADIUS + 802.1X + AES`, и в конце troubleshooting logic.

> [!success] Итог модуля
> Если ты можешь настроить small-site WLAN, объяснить базовый enterprise WLAN на `WLC`, собрать `WPA2-Enterprise` scenario и быстро назвать типичные причины wireless problem, значит Module 13 ты понял правильно.
