# 9.3 Module Practice and Quiz

## Что нужно уметь проговорить без подсказки

После Module 9 ты должен быстро объяснить:

1. почему single default gateway — это single point of failure;
2. что такое `virtual router`;
3. как работает `failover`;
4. чем `HSRP`, `VRRP` и `GLBP` отличаются на уровне идеи;
5. как `priority` и `preemption` влияют на HSRP election.

## Короткий summary

|Тема|Ключевая мысль|
|---|---|
|FHRP|делает default gateway redundant|
|Virtual router|один logical gateway поверх нескольких routers|
|Failover|standby takeover'ит role active router|
|HSRP|Cisco standby-based FHRP|
|VRRP|standards-based alternative|
|GLBP|redundancy + load balancing|

## Формулы для памяти

|Формула|Смысл|
|---|---|
|`Higher priority wins`|кто станет active в HSRP|
|`Equal priority -> higher IP wins`|tie-breaker|
|`Preempt returns preferred router`|router с higher priority может забрать active role обратно|
|`Hello 3 / Hold 10`|default HSRP timers|

## Мини-проверка себя

Попробуй ответить:

- Почему routing convergence сам по себе не спасает hosts с dead default gateway?
- Что именно делает virtual IP в FHRP?
- Почему hosts не замечают failover так сильно, как можно было бы ожидать?
- Что будет, если router с higher priority вернётся, но `preempt` не включён?
- Чем `GLBP` отличается от `HSRP` на уровне идеи?

## Команды для повторения

```bash
standby 1 ip 192.0.2.100
standby 1 priority 150
standby 1 preempt
show standby
show standby brief
```

`standby 1 ip 192.0.2.100` #ciscoIOScommand
Создаёт HSRP virtual IP для group 1.

`standby 1 priority 150` #ciscoIOScommand
Поднимает HSRP priority и делает router более вероятным active router.

`standby 1 preempt` #ciscoIOScommand
Разрешает router'у вернуть себе active role при более высоком priority.

`show standby` #ciscoIOScommand
Показывает подробный HSRP status.

`show standby brief` #ciscoIOScommand
Показывает краткий обзор HSRP groups и roles.

> [!important] Самая важная мысль модуля
> FHRP не меняет routing magically у hosts.
> Он делает так, чтобы hosts вообще не замечали, какой physical router сейчас является их real gateway.

> [!tip] Если повторяешь быстро
> Сначала выучи `virtual router`, потом `HSRP active/standby`, потом `priority + preempt + timers`.

> [!success] Итог модуля
> Если ты можешь объяснить, как host продолжает работать после падения default gateway router благодаря virtual IP/MAC и standby takeover, значит тему FHRP ты понял правильно.
