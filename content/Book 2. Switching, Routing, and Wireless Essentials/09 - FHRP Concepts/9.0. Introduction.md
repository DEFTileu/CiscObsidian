# 9.0 Introduction

До этого модуля сеть уже выглядит “почти отказоустойчивой”: есть switching redundancy, VLANs, routing, dynamic addressing. Но остаётся одна очень неприятная точка отказа — `default gateway`.

Если один router или L3 interface, который является default gateway, падает, hosts на этом LAN теряют выход за пределы своей сети.

Именно это решают `FHRP`.

**FHRP (First Hop Redundancy Protocol)** — группа протоколов, которые обеспечивают резервирование default gateway для hosts. #abbreviation
**first hop** — первый Layer 3 device, к которому host отправляет traffic за пределы своей local network. #networkterm

> [!abstract] Суть модуля
> `FHRP = resilient default gateway`

## Что нужно понять

После темы ты должен уверенно понимать:

- почему обычный single default gateway — это problem;

- чем `HSRP`, `VRRP` и `GLBP` отличаются на уровне идеи;
- как `HSRP` выбирает active и standby routers.
