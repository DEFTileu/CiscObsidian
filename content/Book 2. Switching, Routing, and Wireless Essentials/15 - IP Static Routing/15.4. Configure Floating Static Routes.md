# 15.4 Configure Floating Static Routes

## Floating Static Routes

**floating static route** — backup static route, у которой `Administrative Distance` выше, чем у primary route. #networkterm

Идея:

- пока primary route доступен, floating route не используется;
- если primary path исчезает, backup static route “всплывает” в table.

![](https://itexamanswers.net/wp-content/uploads/2020/06/Floating-Static-Routes.jpg)

## Configure IPv4 and IPv6 Floating Static Routes

Примеры:

```bash
ip route 192.168.1.0 255.255.255.0 10.0.0.2 5
ipv6 route 2001:db8:acad:1::/64 2001:db8:feed:10::2 5
```

`ip route 192.168.1.0 255.255.255.0 10.0.0.2 5` #ciscoIOScommand
Создаёт IPv4 floating static route с `AD 5`.

`ipv6 route 2001:db8:acad:1::/64 2001:db8:feed:10::2 5` #ciscoIOScommand
Создаёт IPv6 floating static route с `AD 5`.

![](https://itexamanswers.net/wp-content/uploads/2020/06/Configure-IPv4-and-IPv6-Floating-Static-Routes.jpg)

## Test the Floating Static Route

Логика проверки:

1. убедиться, что primary route используется;
2. симулировать failure primary path;
3. посмотреть, появилась ли floating route в active table;
4. проверить connectivity ещё раз.

> [!tip] Как быстро запомнить
> `Floating` = route “не активна сейчас”, но готова стать backup.

> [!warning] Exam trap
> Если `AD` floating route будет слишком низким, она может конкурировать с primary route, а не быть backup route.

> [!success] Если понял тему
> Ты понимаешь, как с помощью higher `AD` настроить резервный статический маршрут для IPv4 и IPv6.
