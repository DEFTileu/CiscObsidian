# 15.2 Configure IP Static Routes

Этот раздел уже про конкретные команды, которые чаще всего будут в labs и на exam.

## IPv4 Next-Hop Static Route

Пример:

```bash
ip route 192.168.1.0 255.255.255.0 172.16.2.2
```

`ip route 192.168.1.0 255.255.255.0 172.16.2.2` #ciscoIOScommand
Говорит router: сеть `192.168.1.0/24` искать через next hop `172.16.2.2`.

Такой вариант удобен на Ethernet/broadcast media, но может породить recursive lookup.

## IPv6 Next-Hop Static Route

Пример:

```bash
ipv6 route 2001:db8:acad:1::/64 2001:db8:acad:2::2
```

`ipv6 route 2001:db8:acad:1::/64 2001:db8:acad:2::2` #ciscoIOScommand
Создаёт IPv6 static route через next-hop IPv6 address.

## IPv4 Directly Connected Static Route

Пример:

```bash
ip route 192.168.1.0 255.255.255.0 s0/1/0
```

`ip route 192.168.1.0 255.255.255.0 s0/1/0` #ciscoIOScommand
Создаёт IPv4 static route, указывая только exit interface.

Чаще это логично на point-to-point links.

## IPv6 Directly Connected Static Route

Пример:

```bash
ipv6 route 2001:db8:acad:1::/64 s0/1/0
```

`ipv6 route 2001:db8:acad:1::/64 s0/1/0` #ciscoIOScommand
Создаёт IPv6 static route, где указан только выходной interface.

## IPv4 Fully Specified Static Route

Пример:

```bash
ip route 192.168.1.0 255.255.255.0 s0/1/0 172.16.2.2
```

`ip route 192.168.1.0 255.255.255.0 s0/1/0 172.16.2.2` #ciscoIOScommand
Создаёт fully specified IPv4 static route: и exit interface, и next hop заданы явно.

## IPv6 Fully Specified Static Route

Пример:

```bash
ipv6 route 2001:db8:acad:1::/64 s0/1/0 2001:db8:acad:2::2
```

`ipv6 route 2001:db8:acad:1::/64 s0/1/0 2001:db8:acad:2::2` #ciscoIOScommand
Создаёт fully specified IPv6 static route.

![](https://itexamanswers.net/wp-content/uploads/2020/06/IPv6-Fully-Specified-Static-Route.jpg)

## Verify a Static Route

Главные verify-команды:

```bash
show ip route
show ipv6 route
show ip route static
show ipv6 route static
```

`show ip route` #ciscoIOScommand
Показывает полную IPv4 routing table.

`show ipv6 route` #ciscoIOScommand
Показывает полную IPv6 routing table.

`show ip route static` #ciscoIOScommand
Показывает только IPv4 static routes.

`show ipv6 route static` #ciscoIOScommand
Показывает только IPv6 static routes.

> [!tip] Как быстро запомнить
> `next-hop` = адрес
>
> `directly connected` = interface
>
> `fully specified` = address + interface

> [!success] Если понял тему
> Ты можешь настроить обычные IPv4/IPv6 static routes тремя основными способами и проверить их через routing table.
