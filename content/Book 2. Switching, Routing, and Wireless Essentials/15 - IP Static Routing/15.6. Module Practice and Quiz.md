# 15.6 Module Practice and Quiz

## Что нужно уметь проговорить без подсказки

После Module 15 ты должен быстро объяснить:

1. что такое static route и когда она полезна;
2. чем отличаются `next-hop`, `directly connected` и `fully specified` routes;
3. как настроить IPv4 и IPv6 static routes;
4. как настроить default static route;
5. что такое floating static route;
6. что такое host route;
7. почему в IPv6 с link-local next hop нужен exit interface.

## Короткий summary

|Тема|Ключевая мысль|
|---|---|
|Static route|вручную добавленный маршрут|
|Next-hop route|указан только next-hop IP|
|Directly connected route|указан только exit interface|
|Fully specified route|указаны и interface, и next hop|
|Default route|используется при отсутствии более specific route|
|Floating route|backup path с higher AD|
|Host route|route к одному конкретному host|

## Самые важные термины

**static route** — вручную заданный маршрут. #networkterm
**next hop** — следующее устройство, куда отправляется packet. #networkterm
**exit interface** — интерфейс, через который packet должен выйти. #networkterm
**default static route** — fallback static route. #networkterm
**floating static route** — резервный static route с более высоким `AD`. #networkterm
**host route** — route к одному устройству. #networkterm
**link-local address** — IPv6 address, действующий только на локальном канале. #networkterm

## Команды для повторения

```bash
ip route 192.168.1.0 255.255.255.0 172.16.2.2
ip route 192.168.1.0 255.255.255.0 s0/1/0
ip route 192.168.1.0 255.255.255.0 s0/1/0 172.16.2.2
ipv6 route 2001:db8:acad:1::/64 2001:db8:acad:2::2
ipv6 route 2001:db8:acad:1::/64 s0/1/0
ipv6 route 2001:db8:acad:1::/64 s0/1/0 2001:db8:acad:2::2
ip route 0.0.0.0 0.0.0.0 172.16.2.2
ipv6 route ::/0 2001:db8:acad:2::2
ip route 192.168.1.0 255.255.255.0 10.0.0.2 5
ipv6 route 2001:db8:acad:10::10/128 s0/1/0 fe80::2
show ip route
show ipv6 route
show ip route static
show ipv6 route static
```

`ip route ...` #ciscoIOScommand

`ipv6 route ...` #ciscoIOScommand

## Быстрые формулы

|Формула|Смысл|
|---|---|
|`ip route` = IPv4 static route|базовая команда|
|`ipv6 route` = IPv6 static route|базовая команда|
|`0.0.0.0/0` и `::/0`|default routes|
|`Higher AD = backup route`|floating logic|
|`/32` и `/128`|host routes|

> [!important] Самая важная мысль модуля
> В static routing нужно не просто помнить команды, а понимать, какой именно next-hop model ты выбираешь и почему.

> [!tip] Если повторяешь быстро
> Сначала выучи три вида static route, потом default route, потом floating route, и в конце host routes с `link-local` для IPv6.

> [!success] Итог модуля
> Если ты можешь быстро выбрать правильный syntax для IPv4/IPv6 static route, настроить default и backup route и объяснить логику `/32` и `/128`, значит Module 15 ты понял правильно.
