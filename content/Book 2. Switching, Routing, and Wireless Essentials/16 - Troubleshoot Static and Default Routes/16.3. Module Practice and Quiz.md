# 16.3 Module Practice and Quiz

## Что нужно уметь проговорить без подсказки

После Module 16 ты должен быстро объяснить:

1. как packet проходит через default route, static route и connected route;
2. какие команды использовать для troubleshooting static/default route problems;

4. как отличить missing route от down interface;
5. как методично найти и исправить broken connectivity.

## Короткий summary

|Тема|Ключевая мысль|
|---|---|
|Packet processing|каждый router использует свою local routing logic|
|Default route troubleshooting|проверяй, есть ли route of last resort|
|Static route troubleshooting|проверяй destination, next hop и exit path|
|Interface troubleshooting|`show ip interface brief` быстро показывает status|
|Hop isolation|`traceroute` помогает найти failing segment|
|Neighbor validation|`CDP` помогает подтвердить L1/L2 connectivity|

## Самые важные термины

**extended ping** — ping с дополнительными параметрами, например source address. #networkterm
**next hop** — следующее устройство на пути packet. #networkterm
**default route** — fallback route для unknown destinations. #networkterm
**connected route** — directly attached network route. #networkterm
**CDP (Cisco Discovery Protocol)** — протокол обнаружения соседних Cisco устройств. #abbreviation
**hop** — один этап прохождения packet через router на пути к destination. #networkterm

## Команды для повторения

```bash
ping 192.168.2.1 source 172.16.3.1
traceroute 192.168.2.1
show ip route
show ip route | begin Gateway
show ip interface brief
show cdp neighbors detail
```

`ping 192.168.2.1 source 172.16.3.1` #ciscoIOScommand
Проверяет reachability с конкретного source interface/address.

`traceroute 192.168.2.1` #ciscoIOScommand
Показывает путь packet hop by hop.

`show ip route` #ciscoIOScommand
Показывает routing table и наличие static/default routes.

`show ip route | begin Gateway` #ciscoIOScommand
Быстро подводит к default route и основным route entries.

`show ip interface brief` #ciscoIOScommand
Проверяет addresses и status interfaces.

`show cdp neighbors detail` #ciscoIOScommand
Проверяет, видны ли directly connected Cisco neighbors.

## Таблицы для памяти

|Сравнение|Что запомнить|
|---|---|
|ping vs traceroute|работает ли reachability vs где path ломается|
|show ip route vs show ip interface brief|маршруты vs status interfaces|
|Layer 2 ok vs Layer 3 ok|CDP/neighbor visibility не гарантирует routing correctness|
|Missing route vs Wrong route|route отсутствует vs route есть, но указывает неверно|

## Быстрые формулы

|Формула|Смысл|
|---|---|
|`Ping first`|подтверди problem|
|`Traceroute next`|найди failing hop|
|`show ip route`|проверь наличие path|
|`show ip interface brief`|проверь physical/logical status|
|`CDP visible != routing correct`|L2 ok не равно L3 ok|

> [!important] Самая важная мысль модуля
> Troubleshooting static routes — это не поиск “магической команды”, а последовательная проверка path, route table и interface status.

> [!tip] Если повторяешь быстро
> Запомни порядок: `ping -> traceroute -> show ip route -> show ip interface brief -> show cdp neighbors detail`.

> [!success] Итог модуля
> Если ты можешь быстро локализовать broken hop, проверить default/static route и доказать, проблема в route или в interface, значит Module 16 ты понял правильно.
