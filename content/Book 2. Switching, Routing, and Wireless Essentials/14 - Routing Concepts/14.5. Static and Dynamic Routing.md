# 14.5 Static and Dynamic Routing

Здесь модуль сводит всё вместе: какие route types бывают и как router выбирает лучший path, если вариантов несколько.

## Static or Dynamic?

Нельзя сказать, что один подход “всегда лучше”.

Сравнение:

|Type|Когда хорош|
|---|---|
|Static routing|маленькая и стабильная сеть|
|Dynamic routing|большая, меняющаяся сеть|

### Static Routing

Лучше подходит, когда:

- сеть маленькая;
- topology проста;
- changes редки;
- нужен строгий контроль.

### Dynamic Routing

Лучше подходит, когда:

- много routers;
- routes часто меняются;
- нужна автоматическая адаптация;
- важно уменьшить ручную работу.

## Dynamic Routing Evolution

Dynamic protocols развивались от простых решений к более масштабируемым и быстрым.

![](https://itexamanswers.net/wp-content/uploads/2020/06/Dynamic-Routing-Evolution.jpg)

Модуль подводит к мысли:

- ранние protocols были проще, но менее эффективны;
- более современные protocols лучше считают topology и convergence.

**convergence** — состояние, когда routers пришли к согласованному и актуальному представлению о сети. #networkterm

## Dynamic Routing Protocol Concepts

Для exam полезно запомнить базовые concepts:

## Best Path

Если у router несколько candidate routes, он сравнивает их по определённым rules.

Что важно:

- сначала route должна match destination;
- самый specific prefix выигрывает по longest match;
- если source разный, учитывается `AD`;
- внутри одного protocol может использоваться metric.

**metric** — числовая оценка пути внутри routing protocol, например по hops, cost или bandwidth. #networkterm

![](https://itexamanswers.net/wp-content/uploads/2020/06/14.5.4-Best-Path.gif)

> [!important] Формула выбора
> `Longest match first`
>
> `Then trust source (AD)`
>
> `Then protocol metric if needed`

## Load Balancing

**load balancing** — распределение traffic по нескольким equal-cost paths. #networkterm

Если router знает несколько одинаково хороших routes, он может использовать их параллельно.

Это даёт:

- лучшее использование available paths;
- потенциально лучшую производительность;
- redundancy.
