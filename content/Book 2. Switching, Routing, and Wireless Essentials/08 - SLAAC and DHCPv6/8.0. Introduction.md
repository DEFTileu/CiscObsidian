# 8.0 Introduction

Module 8 продолжает тему dynamic addressing, но уже для IPv6.

Здесь главная сложность не в командах, а в логике: **кто именно выдаёт host'у IPv6 information** и какую роль играют `RA flags`.

**SLAAC (Stateless Address Autoconfiguration)** — механизм, при котором host сам создаёт свой IPv6 GUA на основе информации из router advertisement. #abbreviation
**DHCPv6** — IPv6 version DHCP, которая может выдавать host'у либо часть параметров, либо весь addressing. #networkterm

> [!abstract] Суть модуля
> В IPv6 host почти всегда начинает с `RA`.
> Дальше уже flags в RA подсказывают: использовать `SLAAC`, `stateless DHCPv6` или `stateful DHCPv6`.

## Что нужно понять

После темы ты должен уверенно различать:

- `LLA` и `GUA`;
- `SLAAC only`;
- `stateless DHCPv6`;
- `stateful DHCPv6`;
- `A`, `O`, `M` flags в `RA`;
- `DHCPv6 server`, `client` и `relay agent`.
